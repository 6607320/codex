# quest.html Specification

## 1. Meta Information

- Domain: Scripting
- Complexity: Low
- Language: HTML
- Frameworks: CSS(style.css), JavaScript(app.js) — базовые чары веб-страницы
- Context: Independent Artifact

## 2. Goal & Purpose (Цель и Назначение)

Легенда: quest.html служит витриной Великого Кодекса Техномагии, где открывается вход в мир Летописей испытаний. Это статическая веб-страница, которая формирует интерфейс, связывает стили и скрипты и предоставляет навигацию к Летописи и к репозиторию артефактного сведенья. Зачем нужен этот файл? Чтобы представить кодекс читателю в визуальной оболочке, зарядив плавным переходом к другим частям хроники и к внешним ресурсам.

## 4. Implementation Details (The Source DNA / Исходный Код)

### 4.1. Algorithmic Logic (Для исполняемого кода)

1. Браузер запрашивает файл quest.html как лунный свиток из мира сервера. Сервер возвращает документ, содержащий декларацию типа документа, языковую настройку и связь с ресурсами стилей и скриптов.
2. Внутренний портал страницы загружает таблицу чар — стилевой файл css/style.css и волшебный скрипт js/app.js.
3. Взгляд пользователя встречает навигацию и центральный амулет #app, который будет наполнен будущими интерфейсными ритуалами скриптом.
4. Летопись ока Летописи отображается через ссылку Летопись, а внешний мир — через GitHub — через призрачную дорожку.
5. Язык страницы закреплён за русскоязычным пламем в атрибутах и комментариях, чтобы читатель и браузер понимали сигилы.
6. После загрузки страница остаётся статичной оболочкой, готовой к взаимодействию через внешние скрипты и стили, без дополнительных секретов в тексте.

### 4.2. Declarative Content (Для конфигураций и данных)

Для данного артефакта конфигурационные файлы не входят в состав; активы, которые формируют его поведение, представлены ниже как часть инвентаря артефакта.

## 5. Structural Decomposition (Декомпозиция структуры)

- Документ ДОТЯ: DOCTYPE и корневой html элемент с lang="ru"
- head: meta charset="UTF-8", title, link rel="stylesheet" href="css/style.css"
- body: класс quest-page
  - nav: контейнер навигации
    - div.nav-content: размещение трёх элементов
      - a href="index.html" — Летопись (кнопка-чары)
      - div.nav-title: содержит h1 с названием Великий Кодекс Техномагии
      - a href="https://github.com/6607320/codex" — GitHub ↗
  - div#app — пустой холст, куда призван будет поместиться динамический контент
  - script src="js/app.js" — заклинание, которое оживляет этот мир
- Примечания: комментарии внутри HTML описывают мир и его связи (на уровне Lore)

## 6. System Context & Constraints (Системный контекст и Ограничения)

### 6.1. Technical Constraints

- Performance: Стандартный веб-режим; минимальная задержка за счёт прямой загрузки CSS и JS без тяжёлых зависимостей
- Concurrency: Неприменимо напрямую; страница — одна потоковая загрузка в браузере
- Dependencies: css/style.css, js/app.js, сетевые ссылки для GitHub и внешних ресурсов отсутствуют

### 6.2. Prohibited Actions (Negative Constraints)

- НЕ хранить секреты в явном виде на странице
- НЕ выводить суррогатные данные в консоль в рабочем режиме
- НЕ выполнять синхронные сетевые вызовы в критическом потоке ритуала
- НЕ оборачивать конфигурационные файлы (.yaml, .json) внутрь скриптов
- НЕ менять версии или пути к ресурсам во время реконструкции
