## 2. Goal & Purpose (Цель и Назначение)

Context for Creator: Этот файл служит фронтовым Ритуалом Великий Кодекс Техномагии. Он открывает веб-витрину, где читаемая Летопись встречает стиль и связь репозитория: заголовок, кнопки навигации и портал к GitHub, а затем подготавливает место для исполнения магии JavaScript во внешнем артефакте app.js.

Instruction for AI: Понимай цель как WHY этого артефакта — предоставить статическую веб-страницу-обитель для Летописи и связать её с динамическим поведением через внешнюю ленту скриптов, не загромождая мир лишними элементами.

Описание на русском языке: index.html — это витрина, на которой ритуал визуального оформления сочетается с хранилищем кода: навигация к Летописи, заголовок Великого Кодекса и вход в мир GitHub. Основной путь к динамике задаёт app.js, который оживит каркас страницы без изменения основного строя artefact.

## 4. Implementation Details (The Source DNA / Исходный Код)

### 4.1. Algorithmic Logic (Для исполняемого кода)

1. Ритуал инициализации начинаются с объявления типа документа и языка — doctype и html с языком ru, что задают основу мира.
2. В Скрижали головы вкладываются заклинания кодирования смысла: meta charset UTF-8, заголовок Великого Кодекса Техномагии и привязка к внешнему облику через link к css/style.css.
3. В Кольцо Материального Плана (body) выстраивается Магический Компас (nav) с внутренним контейнером nav-content, где прямо перед глазами читателя рождаются: кнопка-«Летопись», центральное поле навигации с названием Кодекса и ссылка на GitHub.
4. В Сердце пространства создаётся Великий Алтарь app — div с id="app", который станет местом для оживления контента скриптом.
5. Заклинание подключения завершается — подключается внешний Скрипт js/app.js, после чего браузер может запустить логику приложения и наполнить зеркало страницы содержанием, заданным скриптом.
6. Весь процесс выполняется с минимальным вмешательством в разметку, чтобы основная структура оставалась читаемой и предсказуемой, а визуальные эффекты — управляемыми через стили.

### 4.2. Declarative Content (Для конфигураций и данных)

Изначальные данные ограничены: этот артефакт не содержит встроенных конфигураций в явном виде. Все поведение и связь с внешними ресурсами описаны через точки взаимодействия: CSS, JS и внешний репозиторий. В идеале любые параметры, если появляются, определяются через внешние скрипты и сетевые вызовы.

## 5. Structural Decomposition (Декомпозиция структуры)

- DOCTYPE и html (с языком ru) — ядро структуры.
- Head:
  - meta charset="UTF-8" — руническая ясность кодировки.
  - title — название Великого Кодекса Техномагии.
  - link rel="stylesheet" href="css/style.css" — связь с Скрижалью стиля.
- Body:
  - nav — Магический Компас.
  - div.nav-content — внутренний контейнер навигации.
  - a.btn-magic (data-text="Летопись") — кнопка-курок для Летописи.
  - div.nav-title и h1 — центральное имя Кодекса.
  - a (GitHub ↗) — портал к Свитку Исходных Рун.
  - div id="app" — Великий Алтарь, куда будет призвано приложение.
  - script src="js/app.js" — Заклинание призыва Жизненной Силы.

## 6. System Context & Constraints (Системный контекст и Ограничения)

### 6.1. Technical Constraints

- Performance: Standard CPU, минимальная задержка загрузки.
- Concurrency: Асинхронное поведение через внешние скрипты; основной ритуал не блокирует поток.
- Dependencies: css/style.css и js/app.js являются ключевыми соседями; внешний GitHub-портал и ссылки — инфраструктура-опорный след.

### 6.2. Prohibited Actions (Negative Constraints)

- НЕ хранить секреты в открытом тексте; используйте .env для чувствительных данных.
- НЕ печатать сырые данные в консоль в продукции.
- НЕ делать синхронные сетевые вызовы в главном цикле ритуала.
- НЕ оборачивать конфигурационные файлы (.yaml, .json) в сценарии (скрипты).
- НЕ менять версии или пути во время реконструкции артефакта.

   Ожидаемый результат: страница корректно отображается с дефолтной стилистикой браузера и без сбоев, приложение продолжает работать на уровне структуры и JS‑логики.
